Complex JS Input Validtion

Here are some exercises to practice using JavaScript to validate form inputs. First, you will need to implement a .js file somewhere in the page in order to run your validations. Next, read the descriptions and try to write code to satisfy the requirements. You are allowed to make changes to the HTML.

1. Adding up to 100%

In the table below there are 5 inputs that always need to add up to 100. The box labed "Total" should display the current sum of the 5 editable boxes and update when the user changes any of the inputs. If the sum is not 100 after the user changes the inputs then the background of the "tr" element should be given the class ".invalid" which will make it red.

Bonus: What if the user types a non-number into one of the boxes? What are some ways to help them avoid entering non-numbers?

2. Character count

In the text area below the user is allowed to input a minimum of 3 and a maximum of 140 characters. Make the "0 / 140" label update to include the current character count when the user types into the box. If the number of characters is greater than or equal to 3 and less than or equal to 140, the "submit" button should be clickable, otherwise it should stay disabled.

3. Future date

These three dropdowns form a person's birthday. Make sure the person is over 18 years old. The date should be based on your computer's system time. If the person is over 18 make the button clickable, otherwise disabled.

4. Checksum

In the text input the user should enter 9 digits and to be valid, 9th digit should be the sum of the previous 8 digits mod 10. If the input is valid make the submit button clickable.
